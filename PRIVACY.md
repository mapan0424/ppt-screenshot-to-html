# Privacy And Publishing Safety

This project is designed to be open sourced without bundling real presentation material.

## Do Not Commit

- Customer or company slide screenshots.
- Real PPTX files or exported slide images.
- Corporate templates, logos, slogans, watermarks, or proprietary visual systems.
- Names, email addresses, phone numbers, account IDs, or profile images.
- Internal URLs, local filesystem paths, repository names, or project names.
- API keys, access tokens, cookies, credentials, or license files.
- Generated HTML output that still contains private slide text.

## Safe Examples

Use examples that are fully synthetic:

- Fictional company names such as `Example Corp` or `Acme Demo`.
- Self-made base images with no real logos.
- Placeholder business content.
- Public-domain or self-created icons and graphics.

## Release Checklist

Before publishing a release:

1. Run a text scan for private indicators.
2. Manually inspect all image and HTML examples.
3. Confirm that no real source slide is included.
4. Confirm that screenshots are synthetic or explicitly licensed for redistribution.
5. Confirm that generated examples do not preserve private wording from a real deck.

When in doubt, leave the asset out.
